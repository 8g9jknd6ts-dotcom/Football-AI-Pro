## 一、系统概述

### 1.1 目标

通过赛前赔率数据的变化规律，识别"市场过度看好某队但实际该队难以赢球"的比赛。系统不预测比分，不预测让球，只输出"低赔方是否翻车"的概率评估。

### 1.2 输入数据

- 赛前平均赔率（胜/平/负）
- 初始平均赔率（开赛前的初始值）
- 变化统计（多少家机构抬高/降低各方向赔率）
- 总机构数

### 1.3 输出结果

- CRI（冷门风险指数）：量化冷门概率的综合评分
- 分级：极高风险/高风险/中高风险/中等/低风险/极低风险
- 冷门方向概率分布：平局概率 / 客胜（或主胜）概率 / 低赔方赢球概率
- 复核标记：是否触发人工复核条件

## 二、冷门定义

> **冷门 = 赛前被机构一致看好的低赔方，最终未能赢球（平局或输球都算冷门）**

**示例**：
- 赔率 1.22 / 4.22 / 12.23 → 低赔方1.22未赢球（平局4.22或客胜12.23）= 冷门
- 赔率 1.30 / 4.50 / 9.00 → 低赔方1.30输球 = 冷门
- 赔率 2.10 / 3.20 / 3.50 → 无明显低赔方，不纳入分析

**让球不考虑**：系统仅分析胜平负赔率，让球盘口不在分析范围内。

## 三、核心公式（V5.0）

```python
CRI_V5.0 = [ratio_score + delta_score + upset_norm + dds_v2 + eca_v2
           + system_bonus + x1_v2 + suspicion_bonus + c1 + d1_v2 + s3_v2
           + (dds_v2 * 0.5 if System D)]
           × discount_v2 × league_coeff_v2 × consistency_penalty
           × signal_dampening
```

### 3.1 完整计算步骤

#### 第一步：系统分层

| 低赔方赔率 | 系统类型 | 说明 |
|-----------|---------|------|
| ≤ 1.55 | System A（深盘区） | 强队碾压局，需防过热 |
| 1.56 ~ 2.00 | System B（灰色区） | **最佳分析区间** |
| ≥ 2.01 | System C（高赔区） | 实力接近，冷门概率天然高 |
| 平局信号极强 | System D（平局专用） | 平局方向额外加成 |

#### 第二步：ratio_score（市场分歧核心分）

```python
ratio = up_against / down_for  # 抬高低赔方赔率的家数 ÷ 降低低赔方赔率的家数

if ratio > 10:
    ratio_score = 12 + math.log2(ratio/10) * 2
elif ratio > 5:
    ratio_score = 8 + math.log2(ratio/5) * 2
elif ratio > 2:
    ratio_score = 4 + math.log2(ratio/2) * 2
else:
    ratio_score = math.log2(ratio + 1) * 2
```

**关键规则**：
- ratio > 10 是系统唯一高置信度信号来源
- ratio < 5 时冷门判断不可单独作为投注依据
- ratio < 2 基本不可信

#### 第三步：delta_score（赔率变动率）

```python
delta_rate = max((fav_pre - fav_initial) / fav_initial * 100, 0)
delta_score = delta_rate * 0.5
```

#### 第四步：upset_norm（冷门方向支持度）

```python
upset_total = draw_down + upset_down  # 降低平局赔率的家数 + 降低非低赔方向赔率的家数
upset_norm = upset_total / total * 10
```

#### 第五步：dds_v2（平局信号）

```python
dds = draw_down / total * 15
```

**情境重解读规则**（V5.1新增）：
- 同国联赛：DDS正常加权
- 跨大洲杯赛：DDS × 0.5（机构可能在"防范主队小胜"而非看好平局）
- 深盘+强主场（低赔方<1.60且DDS>6）：DDS × 0.3

#### 第六步：eca_v2（极端一致性警报）

```python
if up_against >= 40 and down_for <= 0:
    eca = 15      # 满分警报
elif up_against >= 40 and down_for <= 5:
    eca = 8       # 高度警报
elif up_against >= 35 and down_for <= 0 and system_type == "A":
    eca = 3       # 深盘区中度警报
```

#### 第七步：system_bonus（系统类型加成）

```python
if system_type == "A" and ratio > 5:
    system_bonus = 3
elif system_type == "B" and dds >= 2:
    system_bonus = 2
else:
    system_bonus = 0
```

#### 第八步：x1_v2（过热陷阱）

```python
if fav_pre < 1.35:
    x1_bonus = 5
elif fav_pre < 1.40:
    x1_bonus = 2
else:
    x1_bonus = 0
```

#### 第九步：suspicion_bonus（正路怀疑指数）

```python
suspicion_score = (up_against / total) * 10
suspicion_bonus = 5 if suspicion_score >= 7 else 2 if suspicion_score >= 5 else 0
```

#### 第十步：c1_bonus（平赔过低信号）

```python
league_threshold = {
    "英超": 3.3, "意甲": 3.3, "西甲": 3.4, "德甲": 3.4, "法甲": 3.3,
    "荷甲": 3.5, "德乙": 3.5, "解放者杯": 3.5, "美职": 3.6
}
c1_bonus = 3 if draw_pre < league_threshold.get(league, 3.5) else 0
```

#### 第十一步：d1_v2（平赔下降信号）

```python
d1_delta = draw_initial - draw_pre
d1_bonus = 3 if d1_delta >= 0.15 else 1 if d1_delta >= 0.10 else 0
```

#### 第十二步：s3_v2（赔率漂移，分区权重）

```python
s3_base = 1 if abs(fav_pre - fav_initial) >= 0.05 else 0

if system_type == "A":      # 深盘区
    s3_bonus = s3_base * 0.3
elif system_type == "C":    # 高赔区
    s3_bonus = s3_base * 0.5
else:                        # 灰色区
    s3_bonus = s3_base * 1.0
```

#### 第十三步：System D加成（平局专用）

```python
system_d_bonus = dds * 0.5 if (system_type == "B" and dds >= 2) else 0
```

#### 第十四步：discount_v2（稳胆折扣）

```python
discount = 1.0

# ratio >= 20 时禁用稳胆折扣（反诱盘）
if ratio < 20:
    if system_type == "A":
        if down_for >= 20 and ratio <= 0.2:
            discount = 0.3
        elif down_for >= 15 and ratio <= 0.3:
            discount = 0.5
    else:
        if down_for >= 25 and ratio <= 0.15:
            discount = 0.3
        elif down_for >= 15 and ratio <= 0.25:
            discount = 0.5
        elif down_for >= 8 and ratio <= 0.5:
            discount = 0.7
```

#### 第十五步：league_coeff_v2（联赛系数）

```python
league_coeff_map = {
    "英超": 0.75, "意甲": 0.85, "西甲": 0.85, "德甲": 0.90, "法甲": 0.90,
    "荷甲": 1.0, "荷乙": 1.0, "挪超": 0.80, "瑞典超": 0.95, "芬超": 0.75,
    "日职": 0.85, "日职联": 0.85, "澳超": 0.80, "美职": 1.0, "英冠": 0.85,
    "德乙": 1.0, "沙特联": 1.0, "解放者杯": 1.0, "阿甲": 1.0, "巴甲": 1.0,
    "南美": 1.0, "韩K联": 1.0, "韩K": 1.0, "葡超": 1.0
}
league_coeff = league_coeff_map.get(league, 1.0)
```

#### 第十六步：consistency_penalty（一致性惩罚/封顶）

```python
# V5.1修正：ratio>=20时改为CRI封顶20，而非×1.2
if ratio >= 20:
    consistency_penalty = min(1.0, 20 / base_cri)  # 封顶逻辑
```

#### 第十七步：signal_dampening（信号降噪）

```python
small_leagues = ["芬超", "挪超", "澳超", "日职", "日职联"]
signal_dampening = 0.7 if league in small_leagues else 0.85 if league == "美职" else 1.0
```

#### 第十八步：最终CRI

```python
base_cri = (ratio_score + delta_score + upset_norm * signal_dampening + 
            dds * signal_dampening + eca + system_bonus + x1_bonus + 
            suspicion_bonus + c1_bonus + d1_bonus + s3_bonus + system_d_bonus)

cri = base_cri * discount * league_coeff * consistency_penalty * signal_dampening
```

## 四、系统分层

### 4.1 System A — 深盘区（低赔方赔率 ≤ 1.55）

| 特征 | 说明 |
|------|------|
| 典型赔率 | 1.20 / 5.50 / 12.00 |
| 核心风险 | 深盘过热陷阱（X1信号） |
| S3权重 | 0.3倍（深盘漂移意义不大） |
| 稳胆折扣 | 更严格（down_for≥20才触发） |
| 最佳信号 | ratio > 5 + X1触发 |

### 4.2 System B — 灰色区（1.56 ~ 2.00）

| 特征 | 说明 |
|------|------|
| 典型赔率 | 1.75 / 3.40 / 4.80 |
| 核心风险 | 市场分歧剧烈，方向最难判断 |
| 最佳信号 | ratio > 10 + DDS > 4 |
| 这是系统最有效的区间 | |

### 4.3 System C — 高赔区（≥ 2.01）

| 特征 | 说明 |
|------|------|
| 典型赔率 | 2.20 / 3.20 / 3.10 |
| 核心风险 | 实力接近，冷门基线天然高 |
| S3权重 | 0.5倍 |
| 最佳信号 | ratio > 15 + upset_norm > 3 |

### 4.4 System D — 平局专用

当System B且DDS ≥ 2时，额外增加 `dds × 0.5` 的平局加成。

## 五、信号分量详解

### 5.1 高价值信号（命中率 > 60%）

| 信号 | 触发条件 | 权重 | 说明 |
|------|----------|------|------|
| ratio > 10 | 抬高低赔方赔率家数是降低方的10倍以上 | 12分起 | **系统最强信号** |
| DDS > 6 | >40%机构降低平局赔率 | 最高9分 | 需情境校验（同国联赛有效） |
| ECA = 15 | 40:0或45:0极端一致性 | 15分 | 注意封顶（V5.1） |
| X1触发 | 低赔方赔率 < 1.35 | 5分 | 过热陷阱 |

### 5.2 中等价值信号（命中率 40-60%）

| 信号 | 触发条件 | 权重 | 说明 |
|------|----------|------|------|
| ratio 5-10 | 中度分歧 | 8分起 | 需配合其他信号 |
| S3漂移 | 赔率变动≥0.05 | 0.3-1.0分 | 分区权重 |
| D1平赔下降 | 平赔下降≥0.10 | 1-3分 | 注意深盘场景打折 |
| delta_score | 赔率变动率 | 0-5分 | 剧变时有效 |

### 5.3 低价值/噪声信号（命中率 < 40%）

| 信号 | 问题 | 处理 |
|------|------|------|
| ratio < 2 | 基本不可信 | 忽略或降级 |
| 深盘区S3 | 1.50→1.55无意义 | 权重降至0.3 |
| 小联赛所有信号 | 数据质量差 | 整体×0.7 |

## 八、适用范围与硬性边界

### 8.1 硬性排除（绝对不可用）

- [ ] 低赔方赔率 < 1.20（回报/风险比极差）
- [ ] 低赔方赔率 > 2.80（无明显低赔方）
- [ ] 总机构数 < 30（数据不足）
- [ ] 联赛最后3轮（战意盲区）
- [ ] 杯赛决赛/升级附加赛（战意极端化）
- [ ] 德比战（战意溢价无法量化）
- [ ] 关键球员伤停≥3人（阵容残缺盲区）
- [ ] 赛前7天内换帅（战术剧变）

### 8.2 必须人工复核（可用但需确认）

- [ ] 低赔方赔率 < 1.40 且 ratio > 20
- [ ] CRI > 25
- [ ] 三方向同时有 > 25家机构抬高赔率（GOR全场上调）
- [ ] 低赔方连胜/连败 ≥ 5场
- [ ] 近6次交锋同一结果 ≥ 4次

## 九、已知缺陷与风险提示

### 9.1 不可修复的结构性缺陷

| 缺陷 | 影响 | 缓解方案 |
|------|------|----------|
| **诱盘盲区** | 福伦丹、赫罗纳型翻车 | Bait Signal（V5.1增加赛事阶段校验） |
| **战意盲区** | 完全无法捕捉球队战意 | 硬性排除赛季末/杯赛决赛 |
| **基本面盲区** | 伤停、换帅、天气无法量化 | 人工复核清单 |
| **方向精度低** | 平局55% / 客胜40% / 主胜30% | 输出概率分布，不指定单一方向 |

### 9.2 Bait Signal当前问题

Bait Signal（诱盘识别）在V5.0实战中命中率0%（2次触发2次误报）。**使用建议**：
- 联赛末轮+无欲无求场景：Bait Signal有效
- 杯赛抢分场景：禁用Bait Signal
- 跨级别对决（豪门vs中游）：禁用Bait Signal

## 十一、实战使用流程

```
Step 1: 数据质量检查
  └─ 机构数≥30？赔率1.20~2.80？变化统计完整？
  └─ 不通过 → PASS

Step 2: 排除红线检查
  └─ 联赛末轮？杯赛决赛？德比？换帅？伤停≥3人？
  └─ 命中任一 → PASS

Step 3: 联赛系数调整
  └─ 小联赛×0.7降噪 / 解放者杯×1.0

Step 4: CRI计算
  └─ 按V5.0公式逐条计算

Step 5: 情境校验（V5.1新增）
  └─ 跨大洲杯赛？→ DDS打折 + 客场劣势系数
  └─ 深盘+强主场？→ DDS×0.3
  └─ 触发Bait Signal？→ 检查赛事阶段

Step 6: ratio校验
  └─ ratio<5 → 降级处理
  └─ ratio<2 → 忽略

Step 7: 复核触发检查
  └─ CRI>25？深盘+ratio>20？
  └─ 命中 → 标注"需人工复核"

Step 8: 最终输出
  └─ 分级 + 方向概率分布 + 置信度
```

## 十二、V5.1优化规划

### 12.1 已确定要落地的5项

1. **DDS情境重解读**
   - 跨大洲杯赛：DDS × 0.5
   - 深盘+强主场：DDS × 0.3

2. **Bait Signal赛事阶段校验**
   - 杯赛抢分场景：禁用Bait Signal
   - 跨级别对决：禁用Bait Signal

3. **客场劣势系数**
   - 解放者杯客场：CRI +3
   - 欧冠客场：CRI +1

4. **赔率<1.20与Bait Signal互斥**
   - 赔率<1.20时不再输出Bait Signal

5. **ratio≥20封顶机制**
   - 废除×1.2惩罚，改为CRI封顶20

### 12.2 探索中的方向

- 引入赛前首发名单校验（核心球员轮休识别）
- 引入xG（预期进球）数据交叉验证

- 考虑海拔高度系数（厄瓜多尔、玻利维亚主场）
