# 疯狂红单中文盘口解析协议

版本：Football AI 1.0.0-alpha.36  
状态：ACTIVE（解析协议）；不代表任何盘口模型已获 production 准入。

## 1. 适用范围

适用于国内“疯狂红单”或同类中文赔率截图。截图中的中文盘口描述是原始语义字段，优先级高于任何符号化显示或第三方转写。

## 2. 原始字段登记

每张截图至少登记：

- MatchID、主队、客队、赛事、开赛时间；
- 市场栏目：胜平负、让球胜平负、让球、大小球等；
- 公司名称；
- 初盘/即时盘标记（初、即）；
- 中文盘口原文（例如主让半球、客让半球、受让平/半）；
- 主队水位、客队水位、抓取时间、原图哈希；
- 数据状态：`USER_CERTIFIED_CONTEXT` 或 `SOURCE_AUDITED`。

## 3. 中文语义规范化

中文原文保留不变，同时生成统一字段 `HandicapPerspective`：

| 中文原文 | 统一含义 |
|---|---|
| 主让平/半、主让半球、主让半/一 | 主队让球，按显示的具体级别记录 |
| 客让平/半、客让半球、客让半/一 | 客队让球，按显示的具体级别记录 |
| 受让平/半、受让半球、受让半/一 | 被标注“受让”的一方获得相应让球 |
| 平手、平/半 | 记录为原始级别，不擅自四舍五入 |

“初”与“即”只表示时间截面，不表示方向；方向只能来自同一行的中文盘口原文。

## 4. 冲突与缺失处理

1. 中文标签与符号化盘口冲突时，保留两者并以中文标签为主；冲突计入 `DataQuality=REVIEW`。
2. 初盘或即时盘缺失时，不推断缺失值，标记 `unavailable`。
3. 主客队身份、公司行或市场栏目无法确认时，禁止生成正式让球结论。
4. 只看到水位变化、没有盘口级别变化时，只能描述“水位变化”，不得宣称升盘或降盘。
5. 多家公司必须逐行保留；聚合结果必须同时报告公司数、加权公司数、方向一致率和离散度。

## 5. 输出约束

报告分别输出：竞彩胜平负、竞彩让球胜平负、亚洲盘方向。三者不得互相替代。中文截图可作为单场 `USER_CERTIFIED_CONTEXT`，但不能自动进入长期训练或生产回测，除非完成来源、时间和结果审计。

## 6. 可执行质检

转录后的数据可使用 `scripts/validate_crazy_red_list_cn.ps1` 校验。模板为 `data/templates/crazy_red_list_cn_template.csv`。脚本只负责字段和语义质量门禁，不会把截图数据自动升级为历史训练样本。
