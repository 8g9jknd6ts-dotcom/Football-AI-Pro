# 亚洲盘口回测协议 v1.0

本协议在首次策略回测运行前固定。

## 盘口与结算

- 使用 `ASIAN_HANDICAP_CLOSE / MarketAverage` 的主队盘口、主队赔率与客队赔率。
- `HomeLine` 始终从主队视角表达；客队使用相反方向结算。
- `.25` 和 `.75` 四分之一盘口平均拆为相邻的整数盘与半球盘。
- 支持 `WIN / HALF_WIN / PUSH / HALF_LOSS / LOSS`，按每注1单位计算真实净收益。

## 信息边界

- 模型特征严格使用当前比赛之前的完赛数据。
- 最近窗口10场，双方至少各5场历史。
- 泊松参数固定为主场系数1.15、rho=-0.12、最大比分8球。
- 收盘盘口和赔率只用于当场选择与结算，不进入球队强弱估计。

## 预注册策略

1. 分别计算主队与客队方向的模型期望收益。
2. 只选择期望收益较高的一方。
3. 最高模型期望收益至少为3%，否则跳过。
4. 每次固定下注1单位，不追损、不串关、不按凯利放大。

## candidate → validated 研究门禁

每个联赛必须同时满足：

1. 至少500注；
2. 实际 ROI 大于0；
3. 单注收益均值的正态近似95%置信区间下界大于0；
4. 所有比赛拥有有效 MatchID、盘口、赔率和可复现逐场记录。

即使通过，本回测也不能直接进入 production：使用收盘均价存在实际成交、限额和时间同步偏差，且正态置信区间仅为初步筛选。
