# Football AI Pro 命中率优化协议 v1.0

## 目标

在不牺牲可验证性的前提下，提高正式胜平负推荐的命中率。命中率提升必须同时报告覆盖率、弃权率、联赛分层结果和市场基准差值。

## 预测层

每场先生成完整概率 `P(H), P(D), P(A)`，再进行校准和筛选。AI评分、信心指数只作为筛选特征，不得直接当作概率。

## 推荐门禁

推荐首选必须同时满足：

1. 校准后首选概率达到预注册阈值；
2. 首选概率领先第二选项达到预注册概率差；
3. 与市场基准方向不冲突，或冲突时有独立基本面证据；
4. 数据完整率达到门槛；
5. 联赛和市场处于允许状态；
6. 不触发冷门否决或临场数据缺失门禁。

不满足条件则输出 `ABSTAIN`，不强行给出推荐。

## 小样本处理

联赛参数采用向全球基准收缩的层级方法。每新增一场，只更新下一时间窗口，不回填既有预测。小样本联赛仅允许 SHADOW 记录。

## 验证设计

- 采用按日期排序的滚动扩展窗口；
- 阈值和指标在窗口开始前冻结；
- 每个窗口至少50场，累计至少3个窗口；
- 与同窗口市场最优基准和多数类基准比较；
- 同时报告Accuracy、Brier、LogLoss、Coverage、Abstention、ROI和最大回撤；
- 任何单一窗口改善不得直接升级模型。

## 升级条件

候选模型必须在时间外样本中同时满足：命中率高于市场基准、Brier不恶化、覆盖率不低于预设下限、至少两个联赛或两个独立窗口方向一致，才可进入下一轮候选。

## 禁止

- 不得根据已经看到的结果调整评分阈值；
- 不得把高弃权率造成的高命中率称为全面提升；
- 不得用收盘后信息、赛后阵容或结果修正训练数据；
- 不得把一次串关命中视为模型有效证据。

## 当前诊断警告

在既有评分诊断中，AI评分≥70的筛选样本与市场首选高度重合（多个联赛为100%重合）。因此该阈值目前只能证明“市场强势场次筛选”，不能证明AI产生了独立命中增益。后续必须进行市场剥离实验：分别比较市场首选、AI首选、两者一致和两者冲突四个分层。
