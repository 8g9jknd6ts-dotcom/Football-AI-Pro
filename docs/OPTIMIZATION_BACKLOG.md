# Football AI Pro 优化优先级

版本：Football AI Pro 1.0 / Optimization Backlog v1

## 当前顺序

### alpha.41 后续执行重点

连续多时间点赔率轨迹暂列为数据建设项目，不阻塞当前模型优化。当前优先顺序调整为：

1. 五大联赛归档样本的 MatchID、赛季和结果完整性复核；
2. 五大联赛分别进行欧赔基准概率去水与校准，禁止跨联赛直接合并参数；
3. AI评分与市场基准的分层回测，报告 Brier、LogLoss、覆盖率和弃权率；
4. 冻结弃权阈值，验证“通过/观察/弃权”三档是否改善命中率；
5. 只有完成以上时间外回测后，才调整正式推荐权重。

连续赔率轨迹仍保留为后续增强项；在其缺失期间，盘口走势只能使用已核验的初盘、即时盘或收盘快照，并降低相应信号权重。

1. 小样本影子回测：挪超、巴甲、美职联分别建立滚动扩展窗口，积累时间外样本；未过门禁不得生产推荐。
2. 概率校准：把AI评分和信心指数映射为可验证的胜平负概率，并用Brier、LogLoss和校准曲线检查。
3. 弃权门禁：模型概率差不足、市场信号冲突、数据不完整或临场基本面缺失时，输出“不下注”。
4. 联赛独立参数：分别估计主场优势、平局率、进球分布、赛程和旅行效应，禁止跨联赛直接套参数。
5. 盘口数据标准化：统一球探符号为主队视角；记录初盘、即时盘、封盘、公司、时间和来源哈希；单家公司变化不提高权重。
6. 临场修正：首发、伤停、轮换和动机作为赛前修正层，不能用赛后信息回填历史样本。
7. 让球模型隔离：胜平负、竞彩让球胜平负、亚洲盘分别训练、结算和回测；没有历史亚盘不得宣称亚盘盈利能力。
8. 冷门猎手升级：只作为稳胆否决与风险分层模块，须有跨联赛滚动回测后才能改变主推荐。
9. 串关模型：单场模型通过后，独立回测2串1、3串1的命中率、ROI、回撤和赔率价值。
10. 版本与开发日志：每次参数、规则、数据源和模型变更都记录版本、时间、父版本和回测结果。
11. AI筛选门槛实验：历史诊断显示高AI评分区间命中率更高，但门槛不能事后直接上线；预注册候选门槛为AI评分≥70，并在下一时间窗口验证覆盖率、命中率和稳定性。
12. 评分与赔率解耦检查：高评分若只是在重复市场最短赔率，必须与市场基准分层比较，确认不是重复计分。
13. 市场剥离实验：分别评估市场首选、AI首选、两者一致和两者冲突四类样本；只有AI在冲突层或同市场条件下仍有时间外增益，才允许提高AI权重。
14. 公司级盘口聚合：逐家公司记录初盘/即时盘/时间/盘口/主客水位，计算升降盘计数、降水计数、加权共识、反向公司数和离散度。
15. 来源权重校准：对Crown、Bet365、澳门、威廉及其他公司分别回测其收盘方向信息，未经验证的公司只进入观察层。
16. 公司级数据质检：赔率导入先运行 `scripts/validate_company_odds.ps1`，检查必需字段、价格有效性、重复公司行和比赛唯一性；质检未通过不得解释走势。

## 当前禁止事项

- 不把AI评分直接当成胜率。
- 不把短期命中或三连中当作模型升级证据。
- 不把欧赔市场数据冒充中国竞彩官方赔率。
- 不把截图中的单场上下文样本写入长期训练集。
- 不使用未经历史回测的“诱盘”“伪强化”作为确定性结论。
